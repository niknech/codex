# Telegram Keyword Cleaner (WPF + TDLib, .NET 8)

Приложение для Windows (WPF), которое авторизуется в Telegram через **TDLib** и удаляет сообщения по ключевым словам.

## Возможности

- Авторизация: телефон → код (SMS/Telegram) → 2FA пароль (если включён).
- Хранение TDLib-сессии в `./tdlib_data`.
- Загрузка ключевых слов из `keywords.txt` (UTF-8, 1 слово/фраза на строку).
- Поиск совпадений по подстроке, case-insensitive, с нормализацией:
  - строка приводится к lower-case,
  - `ё` заменяется на `е`.
- Выбор чата:
  - конкретный чат из списка,
  - любой чат,
  - чат по `username` вручную.
- Фильтр по диапазону дат.
- Dry-run режим (ничего не удаляет, только показывает и логирует совпадения).
- Delete-for-all режим (попытка удаления для всех, если возможно; при ошибке запись пропускается).
- Логи операций в UI.
- CSV-лог `deleted_log.csv` с полями:
  - `chat_id, chat_title, message_id, date, matched_keyword, text_snippet`.

## Структура проекта

- `Program.cs` — точка входа WPF.
- `TelegramClientService.cs` — работа с TDLib (инициализация, авторизация, чаты, история, удаление).
- `MessageScanner.cs` — нормализация и поиск ключевых слов.
- `Deleter.cs` — логика dry-run/удаления и обработка ошибок.
- `CsvLogger.cs` — потокобезопасный CSV-логгер.
- Дополнительно:
  - `MainWindow.cs` — UI окна.
  - `AppConfig.cs` — чтение/запись `config.json`.

1. **Windows 10/11 x64**

3. Установленная/подключенная **TDLib**
4. `api_id` / `api_hash` от Telegram

## Где взять `api_id` и `api_hash`

1. Откройте: https://my.telegram.org
2. Войдите по номеру телефона.
3. Зайдите в раздел **API development tools**.
4. Создайте приложение и получите:
   - `api_id`
   - `api_hash`

## Как подключить TDLib в Windows

В проекте добавлен NuGet пакет `TdLib`, но сам нативный TDLib (`tdjson.dll`) должен быть доступен рантайму.

Варианты:

1. Сборка TDLib вручную (официальный репозиторий):
   - https://github.com/tdlib/td
2. Взять готовую Windows-сборку `tdjson.dll` из доверенного источника.

После этого поместите `tdjson.dll`:
- либо рядом с `.exe` (в выходной папке `bin/...`),
- либо в системный путь, доступный приложению.

> Если `tdjson.dll` не найден, приложение не сможет инициализировать TDLib.

## Быстрый старт

1. Создайте файл `config.json` рядом с `.csproj`:

```json
{
  "ApiId": 123456,
  "ApiHash": "your_api_hash_here",
  "PhoneNumber": "+79990000000"
}
```

2. Создайте `keywords.txt` (UTF-8), например:

```text
спам
реклама
badword
```

3. Запуск:

```bash
dotnet restore
dotnet build -c Release
dotnet run -c Release
```

4. В окне приложения:
   - нажмите **Авторизоваться**,
   - введите код и/или 2FA пароль при необходимости,
   - выберите чат/режим,
   - задайте даты,
   - выберите dry-run или реальное удаление,
   - нажмите **Сканировать и удалить**.

## Поведение удаления

- Проверяется текст сообщений и подписи медиа (caption).
- При совпадении:
  - в dry-run: удаление не выполняется, но запись попадает в лог и CSV;
  - в рабочем режиме: вызывается удаление через TDLib.
- В режиме delete-for-all приложение передаёт `Revoke=true`; если TDLib возвращает ошибку, сообщение пропускается (ошибка пишется в лог).

---

## Устойчивость и обработка ошибок

- Ошибки TDLib перехватываются, выводятся в UI-лог.
- Между API-вызовами стоят короткие задержки, чтобы снизить риск ограничений.
- Ошибки удаления отдельного сообщения не останавливают весь проход.

## Файлы данных

- `./tdlib_data` — база и сессия TDLib.
- `keywords.txt` — ключевые слова.
- `config.json` — API и телефон.
- `deleted_log.csv` — итоговый лог совпадений/удалений.

## Важные замечания

- Используйте инструмент ответственно и только там, где имеете право удалять сообщения.
- В публичных/чужих чатах удаление может быть ограничено правами и правилами Telegram.
